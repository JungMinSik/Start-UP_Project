사전 설치 프로그램

1회만 · 전원 필수

개발을 시작하기 전, 아래 4가지를 반드시 설치해야 합니다. 이미 설치되어 있다면 버전만 확인하세요.

STEP A · Node.js
Node.js 20 LTS
Next.js 프론트엔드 실행에 필요합니다. LTS(Long Term Support) 버전으로 설치하세요.
다운로드: nodejs.org → "LTS" 버튼 클릭 후 설치
STEP B · Python
Python 3.11 이상
FastAPI 백엔드 실행에 필요합니다. 설치 시 반드시 아래 옵션을 체크하세요.
⚠ Windows 설치 시 필수 체크 설치 화면 첫 페이지에서 "Add Python to PATH" 체크박스를 반드시 체크해야 합니다. 빠뜨리면 터미널에서 python 명령어를 인식하지 못합니다.
다운로드: python.org → Downloads → Python 3.11.x
STEP C · VS Code
Visual Studio Code
팀 전체가 같은 에디터를 쓰면 확장프로그램, 설정, 단축키가 동일해서 협업이 편합니다.
다운로드: code.visualstudio.com
STEP D · Git
Git (버전 관리)
코드 공유와 협업에 필수입니다. GitHub Desktop을 써도 되지만, 터미널 git도 설치는 해야 합니다.
다운로드: git-scm.com → 운영체제에 맞게 다운로드

설치 완료 후 터미널을 열고 아래 명령어로 버전을 확인하세요:

Terminal — 버전 확인 bash 
node -v      # v20.x.x 가 출력되어야 함
python --version  # Python 3.11.x 가 출력되어야 함
git --version   # git version 2.x.x 가 출력되어야 함

프로젝트 폴더 구조 생성

팀장 1인 실행

팀장 한 명이 아래 명령어로 폴더를 만들고 GitHub에 올려두면, 나머지 팀원들은 07번 Clone 가이드만 따라가면 됩니다.

Terminal — 폴더 생성 bash 
mkdir ai-interviewer
cd ai-interviewer
mkdir frontend backend
완성될 폴더 구조
ai-interviewer/

    backend/  ← FastAPI 프로젝트가 들어갈 곳
// 04

Backend 세팅 (FastAPI)

백엔드 담당자
STEP 1 · 가상환경 (venv)
Python 가상환경 생성 및 활성화
venv는 이 프로젝트만의 독립된 Python 공간입니다. 반드시 만들어야 라이브러리 충돌이 없습니다.
backend/ 폴더 안에서 실행 bash 
cd backend

# 가상환경 생성
python -m venv venv

# 가상환경 활성화 — 운영체제에 맞게 하나만 실행
source venv/bin/activate        # Mac / Linux
venv\Scripts\activate           # Windows PowerShell

# 활성화 되면 터미널 앞에 (venv) 가 붙습니다
(venv) $
⚠ 중요 매번 백엔드 작업을 시작할 때마다 가상환경을 활성화해야 합니다. (venv)가 터미널에 없으면 활성화가 안 된 상태입니다.
STEP 2 · 라이브러리 설치
pip으로 필요한 패키지 설치
가상환경 활성화 상태에서 실행 bash 
pip install fastapi uvicorn sqlalchemy python-dotenv openai pdfplumber python-jose[cryptography] bcrypt python-multipart

# 설치 목록을 파일로 저장 (팀원 공유용)
pip freeze > requirements.txt
STEP 3 · 환경변수
.env 파일 생성 (직접 만들기)
backend/ 폴더 안에 .env 파일을 새로 만들고 아래 내용을 붙여넣으세요. VS Code에서 파일 새로 만들기로 생성하면 됩니다.
backend/.env env 
OPENAI_API_KEY=sk-여기에_본인_OpenAI_키_입력
SECRET_KEY=아무_랜덤_문자열_입력_예시_abc123xyz
⚠ 이 파일은 절대 GitHub에 올리면 안 됩니다 06번 Git 세팅 시 .gitignore에 추가하면 자동으로 제외됩니다. API 키는 팀 단톡으로 별도 공유하세요.
STEP 4 · DB 초기화
SQLite 테이블 자동 생성 (최초 1회)
backend/ 폴더에서 실행 bash 
python -c "from database import Base, engine; Base.metadata.create_all(engine)"
# database.db 파일이 생성됩니다
# VS Code에서 SQLite Viewer로 열어서 테이블 확인 가능
STEP 5 · 서버 실행
FastAPI 개발 서버 시작
backend/ 폴더에서 실행 bash 
uvicorn main:app --reload --port 8000
# --reload : 코드 저장 시 서버 자동 재시작
# 브라우저에서 http://localhost:8000/docs 열리면 성공!
✓ 성공 확인: 브라우저에서 http://localhost:8000/docs 를 열면 FastAPI가 자동 생성한 API 문서(Swagger UI)가 보입니다.
// 05

Frontend 세팅 (Next.js)

프론트엔드 담당자
STEP 1 · 프로젝트 생성
Next.js 14 앱 생성
아래 명령어 하나로 TypeScript + Tailwind + App Router가 자동 설정됩니다.
frontend/ 폴더 안에서 실행 bash 
cd frontend

npx create-next-app@latest . --typescript --tailwind --app --eslint
# 설치 중 질문이 나오면 모두 Enter (기본값) 선택
STEP 2 · 추가 라이브러리
프로젝트에 필요한 패키지 추가 설치
frontend/ 폴더에서 실행 bash 
npm install lucide-react react-dropzone
# lucide-react  : 아이콘 라이브러리
# react-dropzone: PDF 드래그 & 드롭 업로드
STEP 3 · 환경변수
.env.local 파일 생성
frontend/ 폴더 안에 .env.local 파일을 새로 만들고 아래 내용을 입력하세요.
frontend/.env.local env 
NEXT_PUBLIC_API_URL=http://localhost:8000
STEP 4 · 실행
Next.js 개발 서버 시작
frontend/ 폴더에서 실행 bash 
npm run dev
# http://localhost:3000 열리면 성공!
✓ 성공 확인: http://localhost:3000 에서 Next.js 기본 화면이 보이면 완료입니다.

Git 초기 세팅

팀장 1인 실행

팀장이 GitHub에 레포를 만들고 아래 순서대로 진행하세요. 나머지 팀원은 07번으로 건너뛰세요.

ai-interviewer/ 루트 폴더에서 실행 bash 
cd ai-interviewer
git init
git checkout -b dev     # 기본 브랜치를 dev로 설정
.gitignore 생성 · 필수
.gitignore 파일을 루트에 만들기
아래 내용을 ai-interviewer/.gitignore 파일에 붙여넣으세요. 이걸 안 하면 API 키가 GitHub에 올라가는 사고가 납니다.
ai-interviewer/.gitignore gitignore 
# Python
venv/
__pycache__/
*.pyc
*.pyo

# 환경변수 (절대 올리면 안 됨!)
.env
.env.local

# SQLite DB (팀원마다 로컬에서 따로 관리)
*.db

# Node.js
node_modules/
.next/
out/

# 기타
.DS_Store
*.log
첫 커밋 & GitHub 연결 bash 
git add .
git commit -m "feat: initial project setup"

# GitHub에서 레포 만들고 아래 명령어로 연결
git remote add origin https://github.com/팀이름/ai-interviewer.git
git push -u origin dev

브랜치 전략

  • main 브랜치 — 배포 가능한 안정 버전만 merge. 직접 push 금지.
    팀장만 merge 권한을 가집니다
  • dev 브랜치 — 개발 통합 브랜치. 모든 PR은 dev로 보냅니다.
    여기서 팀원 코드가 합쳐집니다
  • feature/기능명 브랜치 — 각자 기능 개발 시 사용. 예: feature/interview-chat
    개발 완료 후 dev로 PR 요청
  • 커밋 메시지 컨벤션: feat: fix: docs: refactor:
    예: git commit -m "feat: 꼬리물기 질문 로직 추가"

팀원 Clone 가이드

팀장 제외 나머지 팀원

팀장이 레포를 만들어 올린 후, 나머지 팀원은 아래 순서만 따라하면 됩니다.

팀원 세팅 전체 순서 bash 
# 1. 레포 클론
git clone https://github.com/팀이름/ai-interviewer.git
cd ai-interviewer

━━━━━ Backend 세팅 ━━━━━
cd backend
python -m venv venv
source venv/bin/activate     # Windows: venv\Scripts\activate
pip install -r requirements.txt
# .env 파일을 직접 생성 (팀장에게 키 받아서 입력)

━━━━━ Frontend 세팅 ━━━━━
cd ../frontend
npm install
# .env.local 파일을 직접 생성
# 내용: NEXT_PUBLIC_API_URL=http://localhost:8000
⚠ .env 파일은 직접 만들어야 합니다 .env 와 .env.local 은 gitignore 되어 있어서 clone 해도 생성되지 않습니다. 팀장에게 OPENAI_API_KEY 값을 받아서 직접 파일을 만드세요.
DB는 팀원마다 로컬에서 따로 관리합니다. database.db도 gitignore 처리되어 있으므로, 04번 STEP 4의 DB 초기화 명령어를 각자 실행해서 빈 DB를 생성하면 됩니다.

매일 작업 시작 루틴

전원 매일

VS Code에서 ai-interviewer/ 폴더 전체를 열고, 내장 터미널을 2개로 분할해서 백/프론트를 동시에 실행하는 게 가장 편합니다.

Terminal 1 · Backend
cd backend
source venv/bin/activate
# Windows:
# venv\Scripts\activate
uvicorn main:app --reload
Terminal 2 · Frontend
cd frontend


npm run dev

자주 하는 실수 체크리스트

  • venv 활성화 안 함
    pip install 전에 꼭 source venv/bin/activate 실행. (venv)가 없으면 전역에 설치됩니다.
  • .env 파일 GitHub에 push
    git add . 전에 .gitignore에 .env가 있는지 반드시 확인. 올라가면 API 키가 노출됩니다.
  • 백엔드 서버 안 켜고 프론트 테스트
    API 호출이 안 되면 uvicorn이 실행 중인지 먼저 확인하세요.
  • main 브랜치에 직접 push
    항상 feature/ 브랜치를 만들고 dev로 PR을 보내세요. main은 직접 push 금지.
  • pip install 후 requirements.txt 업데이트 안 함
    새 패키지를 설치하면 pip freeze > requirements.txt 를 실행하고 커밋하세요.
