# 🤖 AI Interviewer

> AI 기반 모의면접 및 이력서 첨삭 서비스

---

## 📋 전체 흐름

### 1. 모의면접
1. 자기소개서, 이력서, 포트폴리오 업로드
2. 파일 분석
3. 질문 생성 (텍스트 or TTS)
4. 답변 (타이핑 or STT)
5. 3~4번 세션 반복
   - 질문은 꼬리물기 식으로 진행
   - 사용자가 꼬투리 잡힐만한 대답을 하지 않을 때까지
   - 끊는 타이밍은 AI 판단하에 결정
   - 사용자의 요청이 있으면 연장 가능
6. 세션 저장

### 2. 이력서 첨삭
1. 자기소개서, 이력서, 포트폴리오 업로드
2. 텍스트 분석 및 수정 (AI 역할)
3. 완료 후 원문과 수정본 비교 및 수정 이유 서술
4. 최종 결론 피드백 생성 (`~는 더하고 ~는 뺐으면 좋겠고…`)

---

## ⚙️ 기능 정리

### 1. 메인 화면

#### 1) 모의면접 / 이력서 첨삭 버튼
- 버튼 2개 배치
- 모의면접 버튼 클릭 → 모의면접 모드 선택 페이지로 이동 (텍스트 버전 / 이미지 버전 선택)
- 이력서 첨삭 버튼 클릭 → 이력서 첨삭 페이지로 이동

#### 2) 로그인 / 로그아웃 / 기존 대화 확인 버튼
- 로그인 / 로그아웃 버튼 (우측 상단 배치)
- 기존 대화 확인 버튼 → 저장된 세션 목록 페이지로 이동

#### 3) 디자인
- 버튼 2개가 중앙에 크게, 나머지는 최소화
- 군더더기 없는 깔끔한 구성

---

### 2. 모의면접

#### 1) 텍스트 버전

**(1) 파일 업로드**
- 이력서 / 포트폴리오 / 자기소개서 업로드
- 드래그앤드롭 + 클릭 업로드 둘 다 지원
- PDF / DOCX 텍스트 자동 추출 후 프롬프트에 삽입

**(2) 대화 인터페이스**
- AI 말풍선 왼쪽 / 사용자 말풍선 오른쪽 채팅 UI
- 텍스트 입력창 + 전송 버튼
- 마이크 버튼 → STT로 음성 입력 가능 (Web Speech API)
- AI 답변 → TTS로 음성 출력 가능 (Web Speech API)
- 매 답변마다 대화 배열 전체를 OpenAI에 전송해서 꼬리물기 질문 생성

**(3) 종료**
- 종료 버튼 클릭 시 세션 종료
- AI가 스스로 종료 판단하는 로직은 프롬프트로 처리
- 사용자가 연장 요청하면 계속 진행 (프롬프트로 처리)

**(4) 종료 후 피드백**
- 대화 배열 전체를 피드백 프롬프트에 넘겨서 종합 평가 생성
- 잘한 점 / 부족한 점 / 개선 방향 형식으로 출력

#### 2) 이미지 버전

**(1) 파일 업로드**
- 텍스트 버전과 동일

**(2) AI 이미지 인터페이스**
- 면접관 캐릭터 이미지 화면 중앙 배치
- TTS 재생 중에는 캐릭터 말하는 애니메이션 재생 (CSS)
- TTS 종료되면 애니메이션 정지

**(3) 면접관 선택**
- 면접관 이미지 2~3개 중 선택 가능
- 면접관 목소리 선택 가능 (Web Speech API voices 목록 활용)

---

### 3. 이력서 첨삭

#### 1) 텍스트 버전

**(1) 파일 업로드**
- 이력서 / 포트폴리오 / 자기소개서 업로드
- 드래그앤드롭 + 클릭 업로드 둘 다 지원
- PDF / DOCX 텍스트 자동 추출

**(2) 첨삭 인터페이스**
- 추출된 텍스트를 첨삭 프롬프트에 넘겨서 AI가 수정
- AI 응답을 원문 / 수정본 / 수정이유 JSON 형태로 받기
- Before / After 나란히 표시, 수정된 문장 하이라이트
- 수정 이유 툴팁으로 표시

**(3) 종료**
- 사용자가 그만하겠다는 의사 표시 시 종료
- 종료 버튼 명시적으로 배치

**(4) 종료 후 피드백**
- `~는 더하고 ~는 뺐으면 좋겠다` 형식의 종합 의견 출력
- 전체 첨삭 내용 요약 생성

---

### 4. 공통

- 세션 종료 또는 페이지 이탈 시 대화 배열 자동 저장 (localStorage 또는 DB)
- 메인 화면 "기존 대화 확인" 버튼에서 저장된 세션 목록 확인
- 세션 클릭 시 이전 대화 불러와서 이어서 진행 가능
- 모의면접 / 이력서 첨삭 둘 다 적용
